## Project Overview

This project simulates a retail bank's data warehouse with three core tables — `bank`, `customer`, and `transaction_data` — and answers 30 real-world business questions a bank's analytics team might ask, such as:

- Who are our most active and highest-value customers?
- Which branches and regions generate the most revenue?
- What does our investment portfolio look like by type and age group?
- Are there suspicious, overdrawn, or duplicate accounts?
- What are our seasonal/monthly transaction trends?

## Database Schema
![Bank project](https://github.com/saif146/Bank-Customer-Transaction-Investment-Analysis-SQL-Project-/blob/main/erd.png)
### 1. `bank`
Stores branch-level and regional firm performance data.

| Column          | Type             | Description                         |
|-----------------|------------------|--------------------------------------|
| `branch_id`     | INT (PK)         | Unique branch identifier             |
| `city`          | VARCHAR(50)      | City where the branch is located     |
| `region`        | VARCHAR(50)      | Region/zone of the branch            |
| `firm_revenue`  | NUMERIC(20,2)    | Revenue generated by the branch      |
| `expenses`      | NUMERIC(20,2)    | Branch operating expenses            |
| `profit_margin` | NUMERIC(15,2)    | Branch profit margin                 |

### 2. `customer`
Stores customer demographic and branch-assignment data.

| Column           | Type           | Description                                  |
|------------------|----------------|-----------------------------------------------|
| `customer_id`    | INT (PK)       | Unique customer identifier                    |
| `age`            | INT            | Customer's age                                |
| `customer_type`  | VARCHAR(50)    | Type/segment of customer                      |
| `city`           | VARCHAR(50)    | Customer's city                               |
| `region`         | VARCHAR(50)    | Customer's region                             |
| `bank_name`      | VARCHAR(100)   | Name of the bank                              |
| `branch_id`      | INT (FK)       | References `bank.branch_id`                   |

### 3. `transaction_data`
Stores individual transaction and investment records per customer.

| Column                | Type            | Description                                  |
|-----------------------|-----------------|------------------------------------------------|
| `transaction_id`      | INT (PK)        | Unique transaction identifier                 |
| `customer_id`         | INT (FK)        | References `customer.customer_id`             |
| `account_type`        | VARCHAR(50)     | Type of account (e.g., savings, current)       |
| `total_balance`       | NUMERIC(20,2)   | Account balance at time of transaction        |
| `transaction_amount`  | NUMERIC(20,2)   | Amount involved in the transaction            |
| `investment_amount`   | NUMERIC(20,2)   | Amount invested (if applicable)               |
| `investment_type`     | VARCHAR(50)     | Type of investment (e.g., Fixed Deposit)      |
| `transaction_date`    | DATE            | Date the transaction occurred                 |

## Entity Relationship Summary

```
bank (1) ───< (many) customer (1) ───< (many) transaction_data
   branch_id              branch_id          customer_id
                            (FK)                 (FK)
```

- One **bank branch** can have many **customers**.
- One **customer** can have many **transactions/investments**.

## Setup / How to Run

1. **Create the schema** — run `table_create.sql` first. It creates (in order, respecting foreign keys):
   - `bank`
   - `customer` (references `bank`)
   - `transaction_data` (references `customer`)
2. **Load data** into the three tables (not included in this repo — bring your own dataset matching the schema above).
3. **Run the analysis queries** — execute `problem_query.sql` in a PostgreSQL client (e.g., `psql`, DBeaver, pgAdmin).

> ⚠️ This project uses PostgreSQL-specific syntax (`PERCENTILE_CONT`, `EXTRACT`, `INTERVAL '30 days'`, window functions). Minor adjustments are needed for MySQL/SQL Server.

## Analysis Queries (with Code)

### 1. Retrieve All Transactions for a Specific Customer
Get all transactions for customer `204152`, ordered by transaction date ascending.

```sql
select c.customer_id,
       t.transaction_id,
       t.transaction_date,
       t.total_balance,
       t.transaction_amount,
       t.investment_amount
from customer as c
join transaction_data as t
    on c.customer_id = t.customer_id
where t.customer_id = 204152
order by t.transaction_date asc;
```

### 2. Top 5 Customers by Total Number of Transactions

```sql
select t.customer_id,
       count(*) as total_transaction
from customer as c
join transaction_data as t
    on c.customer_id = t.customer_id
group by t.customer_id
order by 2 desc
limit 5;
```

### 3. Calculate the Total Balance for All Accounts

```sql
select sum(total_balance) as total_balance
from transaction_data;
```

### 4. Find Customers with an Investment Amount Greater Than $150,000

```sql
select customer_id,
       sum(investment_amount) as invest_amount
from transaction_data
group by customer_id
having sum(investment_amount) > 150000;
```

### 5. Top 5 Investors Based on Total Investment Amount

```sql
select customer_id, account_type,
       sum(investment_amount) as invest_amount,
       count(*) as times_of_saving,
       round(sum(investment_amount) * 100 / sum(sum(investment_amount)) over(), 2) as pct_of_revenue
from transaction_data
group by 1, 2
order by 3 desc
limit 5;
```

### 6. Identify Duplicate Accounts
Find duplicate customer records based on `customer_id`.

```sql
select customer_id, count(*) as number
from customer
group by customer_id
having count(*) > 1;
```

### 7. Investment Analysis
Average investment plus 25th / 50th / 75th percentile distribution.

```sql
with investment_analysis as (
    select customer_id, sum(investment_amount) as investment_amount
    from transaction_data
    group by customer_id
)

select
    count(*) total_investor,
    round(avg(investment_amount), 2) as avg_investment,
    percentile_cont(.25) within group (order by investment_amount) as investor_25,
    percentile_cont(.50) within group (order by investment_amount) as investor_50,
    percentile_cont(.75) within group (order by investment_amount) as investor_75
from investment_analysis;
```

### 8. Find All Transactions That Occurred in the Last 30 Days

```sql
select *
from transaction_data
where transaction_date > current_date - interval '30 days';
```

### 9. Calculate the Average Investment Amount by Account Type

```sql
select account_type,
       round(avg(investment_amount), 2) as avg_investment_amount
from transaction_data
group by account_type;
```

### 10. Customers Who Did Not Make Any Transaction

```sql
select c.customer_id, t.transaction_date
from customer as c
left join transaction_data as t
    on c.customer_id = t.customer_id
where t.transaction_id is null;
```

### 11. Top 3 Customers Based on Account Type and Transaction Amount

```sql
select customer_id,
       account_type,
       total_amount,
       rnk
from (
    select customer_id,
           account_type,
           sum(transaction_amount) as total_amount,
           rank() over(partition by account_type order by sum(transaction_amount) desc) as rnk
    from transaction_data
    group by customer_id, account_type
) t
where rnk < 4;
```

### 12. Top 5 Customers with the Highest Account Balance

```sql
select customer_id,
       sum(total_balance - transaction_amount) as total_amount
from transaction_data
group by customer_id
order by 2 desc
limit 5;
```

### 13. Top 5 Investors Based on Number of Investments

```sql
select customer_id,
       investment_type,
       count(*) as no_of_invest
from transaction_data
group by 1, 2
order by 3 desc
limit 5;
```

### 14. Customers Who Make Back-to-Back Transactions
Customers whose average gap between consecutive transactions is 1 day.

```sql
with transaction_history as (
    select customer_id,
           round(avg(gap), 2) as avg_of_each_customer
    from (
        select customer_id,
               transaction_date,
               lag(transaction_date) over(partition by customer_id order by transaction_date) as prev_date,
               (transaction_date - lag(transaction_date) over(partition by customer_id order by transaction_date)) as gap
        from transaction_data
    ) sub
    group by customer_id
    order by avg_of_each_customer
)

select *
from transaction_history
where avg_of_each_customer = 1;
```

### 17. Which Branch Makes More Transactions (Count & Amount)

```sql
select c.branch_id,
       count(*) as not_of_transaction,
       sum(t.transaction_amount) as transaction_amount
from transaction_data as t
```

### 18. Which Branch Has More Investment (Count & Amount)

```sql

       count(*) as not_of_investment,
       sum(t.investment_amount) as total_investment_amount
from transaction_data as t
join customer as c
    on t.customer_id = c.customer_id
group by c.branch_id
order by 3 desc, 2 desc;
```

### 19. Find Each Branch's Top Investment Type by Amount

```sql
with investment_info as (
    select c.branch_id,
           t.investment_type,
           sum(t.investment_amount) as total_investment_amount,
           rank() over(partition by c.branch_id order by sum(t.investment_amount) desc) as rnk
    from transaction_data as t
    join customer as c
        on t.customer_id = c.customer_id
    group by 1, 2
)

select *
from investment_info
where rnk = 1;
```

### 22. Percentage of Customers Who Don't Make Transactions, by Region

```sql
with no_transaction_history as (
    select c.customer_id,
           c.region
    from customer as c
    left join transaction_data as t
        on c.customer_id = t.customer_id
    where t.transaction_date is null
),
ranking as (
    select region,
           count(*) as total_customer,
           rank() over(order by region) as rnk
    from no_transaction_history
    group by region
    order by region
),
total_customers as (
    select region as main_region,
           count(*) as total_m_customer,
           rank() over(order by region) as rnk_c
    from customer
    group by region
    order by region
),
result as (
    select r.region,
           round((r.total_customer * 100.0) / t.total_m_customer, 2) as pct_of_total_customer
    from ranking as r
    join total_customers as t
        on r.rnk = t.rnk_c
)

select * from result;
```

### 26. Identify Top Customer(s) by Investment Count

```sql
with investment_info as (
    select customer_id,
           count(*) as no_of_investment,
           rank() over(order by count(*) desc) as rnk
    from transaction_data
    group by customer_id
    order by 2 desc
)

select t.customer_id,
       t.investment_amount,
       t.investment_type
from transaction_data as t
join investment_info as i
    on t.customer_id = i.customer_id
where rnk = 1;
```

### 28. Which Age Group Makes More Investment (Fixed Deposit)

```sql
select
    case
        when c.age between 10 and 30 then '10-30'
        when c.age between 31 and 40 then '31-40'
        when c.age between 41 and 50 then '41-50'
        when c.age between 51 and 60 then '51-60'
        else '70+'
    end as age_status,
    count(*) as number
from customer as c
join transaction_data as t
    on c.customer_id = t.customer_id
where t.investment_type = 'Fixed Deposit'
group by age_status;
```

### 29. Top City of Each Zone Based on Number of Transactions and Amount

```sql
select *
from (
    select b.region,
           b.city,
           count(*) as no_of_transaction,
           sum(t.transaction_amount) as amount,
           rank() over(partition by b.region order by count(*) desc) as rnk
    from transaction_data as t
    join customer as c
        on t.customer_id = c.customer_id
    join bank as b
        on c.branch_id = b.branch_id
    group by 1, 2
) t
where rnk = 1;
```

### 30. Top Transaction Month of Each Year

```sql
select *
from (
    select extract(year from transaction_date) as year_m,
           extract(month from transaction_date) as month_number,
           sum(transaction_amount) as amount,
           rank() over(partition by extract(year from transaction_date) order by sum(transaction_amount) desc) as rnk
    from transaction_data
    group by 1, 2
) t
where rnk = 1;
```

## Key SQL Concepts Used

- **Joins:** `INNER JOIN`, `LEFT JOIN` (anti-join pattern for "no activity" queries)
- **Aggregation:** `SUM`, `COUNT`, `AVG`, `GROUP BY`, `HAVING`
- **Window Functions:** `RANK()`, `LAG()`, `SUM() OVER()`, `PARTITION BY`
- **CTEs (`WITH` clauses):** used for multi-step logic (queries 7, 14, 19, 22, 26)
- **Statistical Functions:** `PERCENTILE_CONT()` for quartile analysis
- **Date Functions:** `EXTRACT()`, `INTERVAL`, `CURRENT_DATE`
- **Conditional Logic:** `CASE WHEN` for age-bucketing
- **Subqueries:** derived tables filtered by `rnk` for top-N-per-group patterns
